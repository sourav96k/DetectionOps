# SOC Analyst L1 Interview Guide (2026)

## Introduction

A Security Operations Center (SOC) Analyst L1 is responsible for monitoring security alerts, investigating suspicious activities, and escalating incidents when necessary.

## 1. What is SIEM?

SIEM stands for Security Information and Event Management. It collects logs from multiple devices, correlates events, and helps analysts identify security incidents.

Examples:

* IBM QRadar
* Splunk
* Microsoft Sentinel

## 2. What is an Offense in QRadar?

An offense is a security incident generated when QRadar correlation rules detect suspicious activity based on events, flows, or threat intelligence.

## 3. What is a Firewall?

A firewall monitors and controls incoming and outgoing network traffic according to security policies.

Examples:

* Palo Alto Networks
* FortiGate
* Sophos Firewall

## 4. What is IDS and IPS?

IDS (Intrusion Detection System):

* Detects suspicious activity
* Generates alerts

IPS (Intrusion Prevention System):

* Detects and blocks malicious activity automatically

## 5. What is Threat Intelligence?

Threat Intelligence is information about cyber threats, attacker techniques, malware campaigns, indicators of compromise (IOCs), and vulnerabilities.

## 6. What is a False Positive?

A false positive occurs when legitimate activity is incorrectly identified as malicious.

## 7. What is a True Positive?

A true positive occurs when an alert correctly identifies malicious activity.

## 8. What is Incident Response?

1. Preparation
2. Identification
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

## 9. What is MITRE ATT&CK?

MITRE ATT&CK is a framework that documents attacker tactics and techniques used during cyber intrusions.

Examples:

* Initial Access
* Execution
* Persistence
* Privilege Escalation
* Defense Evasion

## 10. Common SOC Analyst Responsibilities

* Monitor security alerts
* Investigate suspicious events
* Analyze firewall logs
* Review endpoint alerts
* Escalate incidents
* Create investigation notes

## 11. What is Log Analysis?

Log analysis is the process of reviewing security logs from firewalls, servers, endpoints, and applications to identify suspicious activity.

Examples:

* Failed logins
* Malware detections
* Unauthorized access attempts
* Network scanning activity

## 12. What are Indicators of Compromise (IOCs)?

Indicators of Compromise are artifacts that indicate malicious activity.

Examples:

* Malicious IP addresses
* Malicious domains
* File hashes
* Suspicious URLs

## 13. Common SOC Analyst Tools
